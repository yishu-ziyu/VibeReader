# BDD/TDD: 非线性阅读核心功能（对标 Vibro 超越版）

> 目标：在 VibeReader 上实现 Vibro 声称但源码中未落地的核心功能，并结合 VibeReader 的本地优先优势做差异化。

---

## 功能一：拖拽引用（Drag-to-Inject）

### 业务背景
当前 VibeReader 的选区注入需要"划选 → 点击注入按钮 → 确认"三步。Vibro 的拖拽引用只需"划选 → 拖拽到 AI 面板"一步，交互更自然。

### BDD 行为用例

#### 用例 1.1：从阅读器拖拽文本到 AI Chat
- **Given** 用户在左侧阅读器中选中了文本
- **When** 用户按住选中文本并拖拽到右侧 AI Chat 面板
- **Then** AI Chat 输入框中出现带有来源标记的引用文本
- **And** 用户可以直接在引用后追加问题并发送

#### 用例 1.2：拖拽高亮批注到 AI Chat
- **Given** 用户之前在某段落创建了高亮批注
- **When** 用户从批注列表中拖拽该批注到 AI Chat
- **Then** AI Chat 输入框中出现该批注的文本内容和页码来源

#### 用例 1.3：拖拽取消
- **Given** 用户开始拖拽选中文本
- **When** 用户在非 AI Chat 区域释放鼠标
- **Then** 拖拽操作取消，不触发任何注入

### 验收标准
- [ ] 选中文本支持 HTML5 Drag and Drop API
- [ ] AI Chat 面板支持 `dragover` 和 `drop` 事件
- [ ] 拖拽释放后，输入框中出现格式化引用（`> 引用文本 [P{页码}]`）
- [ ] 拖拽过程中有视觉反馈（半透明遮罩 + 提示文字）
- [ ] 浏览器和 Tauri 桌面环境均可用

## 功能二：段落级非线性思维树（True Thinking Tree）

### 业务背景
当前 VibeReader 的 MindMap 只是"章节标题树"，没有内容解构。真正的思维树需要对每个段落生成一句话摘要，形成可展开/折叠的层级结构。

### BDD 行为用例

#### 用例 2.1：生成段落级思维树
- **Given** 用户打开了一篇 PDF 论文
- **When** 用户点击"生成思维树"按钮
- **Then** AI 对论文每个段落生成一句话摘要
- **And** 树状结构展示：论文标题 → 章节 → 段落摘要 → 要点

#### 用例 2.2：展开/折叠节点
- **Given** 思维树已生成
- **When** 用户点击某个章节节点的展开/折叠按钮
- **Then** 该章节下的段落摘要展开或折叠

#### 用例 2.3：点击节点跳转原文
- **Given** 思维树已生成
- **When** 用户点击某个段落摘要节点
- **Then** PDF 阅读器滚动到该段落所在位置
- **And** 该段落文本被临时高亮（3 秒后消失）

### 验收标准
- [ ] 思维树不是章节标题树，而是段落级内容解构
- [ ] 每个段落摘要 ≤ 50 字
- [ ] 树节点支持展开/折叠
- [ ] 点击节点跳转原文对应段落
- [ ] 生成过程中显示进度指示器
- [ ] 大文件（>50页）分段生成，避免超时

## 功能三：双向锚定（Bidirectional Anchor）

### 业务背景
思维树和原文之间需要双向映射：从思维树可以跳回原文，从原文可以定位到思维树节点。

### BDD 行为用例

#### 用例 3.1：思维树 → 原文锚定
- **Given** 思维树已生成
- **When** 用户点击思维树中的段落节点
- **Then** PDF 跳转到该段落
- **And** 段落文本被脉冲高亮（fade in/out 两次）

#### 用例 3.2：原文 → 思维树锚定
- **Given** 思维树已生成，用户正在阅读 PDF
- **When** 用户点击 PDF 中的某个段落（或该段落的高亮批注）
- **Then** 思维树自动展开到对应节点
- **And** 该节点被滚动到可视区域并高亮

#### 用例 3.3：批注与思维树联动
- **Given** 用户在段落 A 创建了批注
- **When** 用户在思维树中查看段落 A 的节点
- **Then** 节点上显示批注数量徽章
- **And** 点击徽章可以查看批注内容

### 验收标准
- [ ] 思维树节点与原文段落有稳定的 ID 映射
- [ ] 双向跳转延迟 < 300ms
- [ ] 跳转后有视觉反馈（高亮/脉冲）
- [ ] 批注与思维树节点双向可见

## 功能四：AI 注意力导航仪（Attention Navigator）

### 业务背景
Vibro 声称 AI 是"注意力导航仪"，但源码中没有实现。VibeReader 可以率先实现：AI 主动分析论文，标记关键位置。

### BDD 行为用例

#### 用例 4.1：AI 预分析标记关键位置
- **Given** 用户打开了一篇 PDF
- **When** 用户点击"AI 分析"按钮
- **Then** AI 输出 3-5 个"关键洞察点"
- **And** 每个洞察点包含：类型（创新点/方法亮点/实验反常/关键对比）、位置（页码+段落）、一句话说明

#### 用例 4.2：阅读器中显示导航标记
- **Given** AI 分析已完成
- **When** 用户查看 PDF 阅读器
- **Then** 关键位置旁边出现小旗帜图标
- **And** 不同类型用不同颜色（创新点=绿色，方法亮点=蓝色，实验反常=橙色，关键对比=紫色）

#### 用例 4.3：点击标记展开解释
- **Given** 阅读器中显示了导航标记
- **When** 用户点击某个标记
- **Then** 弹出浮动卡片显示 AI 的解释
- **And** 提供"深入阅读"和"跳过"两个按钮

### 验收标准
- [ ] AI 分析输出结构化 JSON（类型、位置、说明）
- [ ] 标记在 PDF 侧边栏或页边显示
- [ ] 标记类型有颜色区分
- [ ] 点击标记弹出浮动卡片
- [ ] "深入阅读"按钮触发 AI 详细解释
- [ ] 分析结果缓存，切换文档不丢失

## 技术实现要点

### 段落级文本提取
- 复用现有的 `pdfService.extractTextFromPDF`，但需要段落级粒度
- 使用 pdf.js 的 `textContent.items` 按位置（y 坐标）分组为段落
- 每个段落生成唯一 ID（`page-index-start-end`）

### AI 提示词设计

**思维树生成提示词：**
```
你是学术阅读助手。请对以下论文段落生成结构化摘要。

要求：
1. 对每个段落生成一句话核心摘要（≤50字）
2. 识别段落类型：背景/方法/实验/结果/讨论/其他
3. 输出 JSON 格式：
   {
     "sections": [
       {
         "title": "章节标题",
         "paragraphs": [
           {"id": "p-1-0", "summary": "摘要", "type": "方法"}
         ]
       }
     ]
   }
```

**注意力导航提示词：**
```
你是学术阅读助手。请分析以下论文，找出最值得读者关注的位置。

要求：
1. 找出 3-5 个关键洞察点
2. 每个洞察点包含：
   - type: "innovation" | "method" | "anomaly" | "comparison"
   - location: {"page": 1, "paragraph": 2}
   - description: "一句话说明为什么这个位置重要"
3. 输出 JSON 数组
```

### 状态存储
- 思维树数据存储在 `documentStore` 中，按 documentId 隔离
- AI 分析结果缓存，避免重复生成
- 锚定映射表：`paragraphId ↔ treeNodeId`

## 优先级

| 优先级 | 功能 | 原因 |
|-------|------|------|
| P0 | 拖拽引用 | 交互优化，降低使用门槛，Vibro 已实现 |
| P0 | 段落级思维树 | 核心差异化，Vibro 声称但未实现 |
| P1 | 双向锚定 | 思维树的必要配套，提升体验 |
| P1 | AI 注意力导航仪 | 超越 Vibro 的独有功能 |
