# PDF 划词体验修复 BDD

日期：2026-06-02

## 行为规则

### B1：划词优先于段落点击

Given 用户在 PDF 正文上按住并拖拽
When 鼠标释放后形成文本选区
Then 系统应保留该文本选区并显示选区操作条
And 不应把这次拖拽误判为段落点击或 Skim Map 导航

业务规则：读者正在“摘一句话”，不是在切换阅读焦点。

### B2：跨行和反向选择必须稳定

Given 用户从同一页 PDF 的任意文字开始选择
When 选区跨多个 text span、跨行，或从右往左/从下往上反向拖拽
Then 只要选区与 PDF text layer 有交集，就应被识别为 PDF 选区

业务规则：PDF text layer 的内部拆分不应暴露给用户。

### B3：操作条不打断继续选择

Given 用户已经划中一段文字
When 操作条出现
Then 操作条应贴近当前阅读滚动区并保持在可见范围内
And 用户在操作条内输入笔记时不应清空当前选区

业务规则：操作条是选中后的后续动作，不应反向破坏选中动作。

## 边界

- 本切片不自动调用 LLM。
- 本切片不改变 Lens Card、注入 AI、高亮、保存笔记的业务入口。
- 本切片不重写 PDF.js text layer，只修正当前透明 text span 的交互策略。
