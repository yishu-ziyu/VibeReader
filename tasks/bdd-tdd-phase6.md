# Phase 6 BDD/TDD：PDF 大纲与最小批注

最后更新：2026-05-23

## 行为规格

### 行为 1：PDF 大纲可显示并跳转

Given 用户打开一个带目录/书签的 PDF
When PDF 阅读器读取 `pdfDoc.getOutline()`
Then 左侧阅读器应显示大纲条目，点击条目后跳转到对应页码

业务规则：阅读器必须支持长文档快速导航，否则只能算 PDF 预览器。

### 行为 2：无大纲 PDF 不阻塞阅读

Given 用户打开一个没有目录/书签的 PDF
When `pdfDoc.getOutline()` 返回空值或空数组
Then 阅读器不应报错，仍保留翻页、缩放、选区注入能力

业务规则：很多论文没有 PDF bookmark，目录能力不能破坏基础阅读闭环。

### 行为 3：选区可创建高亮记录

Given 用户在 PDF 文本层选中一段文字
When 用户点击“高亮”
Then 系统应保存一条批注记录，包含文档 ID、页码、选中文本、颜色、创建时间

业务规则：第一版批注不要求写回 PDF 文件，但必须能把阅读痕迹保存下来。

### 行为 4：选区可创建笔记记录

Given 用户在 PDF 文本层选中一段文字
When 用户输入笔记并保存
Then 系统应保存一条带 note 字段的批注记录

业务规则：AI 阅读器必须让用户沉淀自己的理解，不只是把文本发给模型。

### 行为 5：批注刷新后仍可读取

Given 用户已经创建过 PDF 批注
When 应用重新加载或重新读取批注列表
Then 该文档的批注列表仍应包含之前保存的记录

业务规则：批注是用户资产，必须持久化。

## 边界条件

- 第一版不把高亮写回 PDF 文件。
- 第一版不保证重开后在 PDF canvas 上精确复原 overlay 位置；批注列表、页码、选中文本、颜色、笔记必须可靠。
- 大纲跳转先支持可解析到页码的 destination；解析失败的条目禁用或无动作，不阻塞阅读。
