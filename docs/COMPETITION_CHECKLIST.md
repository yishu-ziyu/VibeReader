## 风险预案

### 风险 1：AI 网络请求失败
**现象**：注入后一直 loading，无响应
**应对**：
1. 等待 3 秒后说"网络有点波动，我展示预设的回答效果"
2. 切换到 `demo-fallback-answer.md` 中准备好的回答文本
3. 继续后续功能演示

### 风险 2：PDF 渲染慢
**现象**：PDF 加载超过 5 秒
**应对**：
1. 提前在后台打开应用并加载好 PDF
2. 或者使用 `wonderland_short.pdf`（小文件，加载快）

### 风险 3：Kimi 免 Key 失效
**现象**：AI 返回 401 / 403
**应对**：
1. 切换到 MiniMax Token Plan（已预配置）
2. 或直接使用 fallback 回答

### 风险 4：演示超时
**现象**：时间超过 3 分钟
**应对**：
1. 跳过非核心功能（导图/闪卡可略）
2. 直接进入结语

## 评委可能的问题

### Q1: 和 ChatPDF / SciSpace 有什么区别？
**A**: 三个核心差异：① 本地渲染零上传，保护隐私；② 多格式支持（PDF+MD+HTML+TXT）；③ 学习闭环——不只是问答，还能生成摘要、闪卡、思维导图，让你"读完还能带走"。

### Q2: 商业模式是什么？
**A**: 短期免费开源获取用户，中期通过团队协作和高级 AI 功能收费，长期构建个人知识图谱的 SaaS 服务。

### Q3: 技术难点是什么？
**A**: 三个技术难点：① Tauri 原生 HTTP 绕过浏览器 CORS 限制，生产环境稳定通信；② pdf.js 文本解析与 Canvas 渲染的坐标对齐，实现精确高亮；③ 多文档状态隔离，8 个独立 Zustand Store 防止切换文档时状态串扰。

### Q4: 团队几个人？开发了多久？
**A**: 1 人全栈开发，7 天完成从 Tauri 壳到完整功能迭代，包含 66 个单元测试和 20 个 E2E 测试。

### Q5: 如何保护用户隐私？
**A**: 本地优先架构——PDF 用 pdf.js 本地渲染，文件不上传；AI 通信通过用户自有的 API Key 直连厂商，我们不触碰用户数据。

### Q6: 支持离线吗？
**A**: 阅读器完全离线可用。AI 功能需要联网（用户自有 Key），但阅读、批注、大纲导航、已生成的摘要/闪卡/导图全部本地存储，断网可用。
