# VibeReader 比赛路演 PPT 大纲

## Slide 1: 封面（5秒）
- 标题：VibeReader —— 你的 AI 阅读实验室
- 副标题：从"读完即忘"到"读有所成"
- 一句话定位：本地优先的学术阅读 Agent，让每一篇论文都变成你的知识资产

## Slide 2: 痛点故事（30秒）
**场景：** 研一学生小林，周五导师丢了 10 篇论文，下周一要汇报

**5 大崩溃时刻：**
1. 满屏公式，看了 3 遍不懂在推什么
2. 专业术语堆积，每段查 10 个词
3. 方法部分读了 N 遍，实验设计还是云里雾里
4. 读完一篇忘一篇，笔记散落在 5 个软件里
5. 想对比多篇观点，在 10 个标签页切到崩溃

**数据支撑：**
- 全球 2000 万+ 科研人员
- 年均阅读 200+ 篇论文
- 80% 的阅读时间浪费在"理解"而非"思考"

## Slide 3: 现有方案为什么不行（20秒）

| 产品 | 模式 | 问题 |
|------|------|------|
| ChatPDF | 上传 → 问 → 得到答案 | 一次性消费，看完即走 |
| SciSpace | 在线阅读 + AI 辅助 | 论文上传云端，隐私风险 |
| Zotero | 文献管理 + 阅读 | 没有 AI，阅读体验停留在 2010 |

**核心矛盾：** 现有工具帮你"读完"论文，但没有帮你"读懂"和"记住"论文。

## Slide 4: VibeReader 解决方案（30秒）

**核心理念：从"论文消费者"到"知识生产者"**

**三层架构：**
1. **阅读层**：PDF + Markdown + HTML + Text 通用阅读器，本地渲染，零上传
2. **AI 层**：多模型路由（Kimi / MiniMax / Claude / OpenAI），选中文本即问即答
3. **学习层**：结构化摘要 → 闪卡 → 思维导图，读完带走一套学习资料

**差异化关键词：本地优先、多格式、学习闭环**

## Slide 7: 技术亮点（30秒）

**技术栈：**
- Tauri v2 + React 18 + Vite 5 → 桌面应用 5MB 体积
- pdf.js 本地渲染 → 零上传，隐私保护
- 多模型流式路由 → 兼容 OpenAI / Anthropic / 国产模型
- Zustand 状态管理 → 8 个独立 Store，状态完全隔离

**技术效率：**
- 首屏加载 458KB（代码分割优化，从 2.29MB 降至 458KB）
- Tauri 原生 HTTP → 生产环境零 CORS 问题
- 48 个单元测试 + 20 个 E2E 测试 → 全自动化 QA

## Demo 脚本（3 分钟严格版）

### 准备
- 提前打开 VibeReader
- 加载 `demo-assets/outline-demo.pdf`
- 配置 Kimi 免 Key（确保网络通畅）

### 脚本

**[0:00-0:05] 开场**
"大家好，我是 VibeReader 的开发者。今天我要展示的不是一个聊天机器人，而是一个能让你'读完还能带走'的 AI 阅读实验室。"

**[0:05-0:15] 零配置打开**
（操作：打开应用）
"打开应用，不需要配置 API Key，不需要注册账号。Kimi 免 Key 体验，开箱即用。"

**[0:15-0:30] 加载论文**
（操作：拖拽 outline-demo.pdf 到左侧）
"这是一篇带大纲的学术论文。左侧是阅读器，右侧是 AI 助手。注意看，大纲导航条自动出现了。"

**[0:30-0:45] 大纲导航**
（操作：点击 Methods 大纲项）
"点击 Methods，直接跳转到第 2 页。长论文不再迷路。"

**[0:45-1:05] 划选注入**
（操作：划选 Introduction 段落，点击"注入 AI"）
"看到不懂的地方，直接划选，点击'注入 AI'。AI 的回答基于论文上下文，不是通用回答。"

**[1:05-1:25] AI 回答**
（等待 AI 回答，展示 LaTeX 公式渲染）
"注意 AI 的回答中，公式是用 LaTeX 渲染的，这是学术阅读的关键体验。"

**[1:25-1:45] 批注高亮**
（操作：选中文本，点击"高亮"，再选一段保存笔记）
"阅读过程中，随时保存高亮和笔记。关闭应用后重新打开，高亮还在原位。"

**[1:45-2:05] 结构化摘要**
（操作：切换到 Summary Tab，点击生成）
"不只是聊天。AI 能主动分析论文的 IMRAD 结构，生成结构化摘要。3 分钟把握全文。"

**[2:05-2:25] 闪卡生成**
（操作：切换到 Flashcards Tab，点击生成）
"读完论文，一键生成闪卡。Q&A 形式的记忆卡片，帮你记住核心概念。"

**[2:25-2:45] 思维导图**
（操作：切换到 Mind Map Tab，点击生成）
"思维导图把论文结构可视化。Introduction → Methods → Results → Discussion，一目了然。"

**[2:45-3:00] 多文档切换**
（操作：打开 sample.md，再切回 PDF）
"VibeReader 不只是 PDF 阅读器。Markdown、Text、HTML 都能读。切换文档时，状态完全隔离，不会串页。"

**[3:00] 结语**
"这就是 VibeReader —— 你的 AI 阅读实验室。从'读完即忘'到'读有所成'。谢谢！"

## 风险预案

### 风险 1：AI 网络请求失败
**应对：** 提前测试网络，准备 `demo-fallback-answer.md` 作为备用回答。

### 风险 2：PDF 渲染慢
**应对：** 使用 outline-demo.pdf（小文件），提前预加载。

### 风险 3：Kimi 免 Key 失效
**应对：** 准备 MiniMax Token Plan 作为备用配置。

### 风险 4：演示超时
**应对：** 严格计时每个环节，超时则跳过非核心功能（导图/闪卡可略）。
